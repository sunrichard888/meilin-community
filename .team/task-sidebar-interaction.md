# Task: 修复首页侧边栏模块的交互性问题

## 问题描述

用户反馈首页侧边栏的三个模块都是"静态"的，无法点击跳转：

### 1. 社区概况 (Community Stats)
- **现状**: 4 个统计数字（活跃邻居、今日发帖、正在进行活动、认证商家）都是纯文本，无法点击
- **期望**: 整个卡片或每个统计项可点击，跳转到 `/community-stats` 页面

### 2. 社区公告 (Announcements)
- **现状**: 3 个公告项无法点击，没有"查看更多"入口
- **期望**: 
  - 每个公告项可点击，跳转到 `/announcements` 页面
  - 底部添加"查看更多公告"链接

### 3. 热门话题 (Hot Topics)
- **现状**: 话题链接是假的 `/topic/${i}`，无法跳转到有意义的页面
- **期望**: 
  - 每个话题可点击，跳转到 `/hot-topics` 页面（或带话题参数的 feed 页面）
  - 底部添加"查看更多话题"链接

## 文件位置

- **需要修改**: `src/components/side-bar.tsx`
- **已有页面**: 
  - `/community-stats` (已存在)
  - `/announcements` (已存在)
  - `/hot-topics` (已存在)

## 技术要求

1. 使用 Next.js Link 组件实现客户端导航
2. 保持现有视觉设计不变
3. 添加 hover 效果提示可点击
4. 确保移动端响应式正常
5. 符合 44px 最小触摸目标

## 验收标准

- [ ] 社区概况卡片整体可点击，跳转到 `/community-stats`
- [ ] 社区公告每个条目可点击，跳转到 `/announcements`
- [ ] 社区公告底部有"查看更多"链接
- [ ] 热门话题每个条目可点击，跳转到 `/hot-topics`
- [ ] 热门话题底部有"查看更多"链接
- [ ] 所有链接有清晰的 hover 反馈
- [ ] TypeScript 编译通过
- [ ] 移动端布局正常

## 注意事项

- 这是纯前端交互优化，不涉及后端数据
- 保持代码简洁，不要过度设计
- 遵循项目现有的代码风格
