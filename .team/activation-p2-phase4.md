## 搜索功能需求

### 搜索范围
1. **搜索帖子**
   - 按内容全文搜索
   - 按作者搜索
   - 按小区搜索

2. **搜索用户**
   - 按昵称搜索
   - 按小区筛选

3. **搜索结果**
   - 分页显示
   - 高亮关键词
   - 排序（相关性/时间）

### 技术选型

#### 方案 A: PostgreSQL 全文搜索（推荐）
**优点**:
- 无需额外服务
- 与现有数据库集成
- 支持中文分词（需配置）
- 成本低

**缺点**:
- 中文分词需要额外配置
- 搜索性能一般（万级数据内 OK）

**实现**:
```sql
-- 使用 tsvector + tsquery
CREATE INDEX posts_search_idx ON posts USING GIN(to_tsvector('simple', content));
```

#### 方案 B: Meilisearch/Algolia
**优点**:
- 搜索性能优秀
- 支持拼音、错别字
- 开箱即用

**缺点**:
- 需要额外服务
- 增加成本
- 集成复杂度高

### 决策
采用**方案 A: PostgreSQL 全文搜索**
- 当前数据量不大，性能足够
- 降低复杂度和成本
- 后续可平滑迁移到专业搜索引擎
