# Task: 首页分类标签筛选功能

## 问题描述

首页的分类标签按钮（全部、紧急通知、二手闲置、邻里互助、社区活动、宠物交友、美食分享）目前：
- ❌ 点击无反应
- ❌ 没有筛选功能
- ❌ 帖子数据是静态假数据

## 目标

实现完整的分类筛选功能：
1. 点击分类标签可筛选对应类别的帖子
2. 从数据库获取真实帖子数据（非静态假数据）
3. 支持按分类字段过滤
4. 保持"全部"显示所有帖子

## 分类定义

| 分类 | 标识 | Emoji |
|------|------|-------|
| 全部 | all | - |
| 紧急通知 | emergency | 🚨 |
| 二手闲置 | marketplace | 🏪 |
| 邻里互助 | help | 🆘 |
| 社区活动 | event | 🎉 |
| 宠物交友 | pets | 🐕 |
| 美食分享 | food | 🍳 |

## 技术方案

### 1. 数据库扩展
- 为 `posts` 表添加 `category` 字段（TEXT 类型）
- 添加索引优化查询性能
- 更新发帖 API 支持传入 category

### 2. 前端实现
- `usePosts` hook 添加 category 参数支持
- 首页添加分类状态管理
- 分类标签按钮添加点击事件和选中样式
- 根据选中分类过滤帖子列表

### 3. 发帖功能
- PostComposer 组件添加分类选择
- 发帖时自动保存分类信息
