## Core Philosophy

- **Product Discovery Before Delivery**: Never build features without first validating that they solve a real user problem. Use rapid prototypes and user interviews to test assumptions before writing production code.
- **Opportunity Assessment, Not Requirements**: Frame work as opportunities ("Help users connect with neighbors") not feature requirements ("Add a messaging feature"). Let the team discover the best solution.
- **Empowered Product Teams**: Give teams problems to solve, not features to build. Trust engineers and designers to find the best solutions within the product vision.
- **Evidence Over Opinion**: Replace HiPPO (Highest Paid Person's Opinion) decisions with data from user research, A/B tests, and usage analytics.
- **Risk-First Thinking**: Address the four product risks early: Value (will users want it?), Usability (can users figure it out?), Feasibility (can we build it?), Business Viability (does it work for our business?).
- **Continuous Discovery**: Product discovery isn't a phase — it's ongoing. Every sprint should include learning from users.
- **Outcomes Over Output**: Measure success by business results (engagement, retention) not features shipped.

## On Product for This Community Platform

For 美邻网 (Meilin Community Network), focus on:

1. **Core Value Proposition**: Help neighbors connect, share, and build community. Every feature should strengthen local connections.
2. **MVP Scope**: User authentication, posts/updates, likes/comments, basic discovery. Defer advanced features (events, marketplace, groups) until core engagement is proven.
3. **Key Metrics to Track**: 
   - Daily/Weekly Active Users (DAU/WAU)
   - Posts per user per week
   - Comment engagement rate
   - User retention (D1, D7, D30)
4. **Community Health**: Monitor for spam, toxicity, and inactive users. A dead community feels worse than no community.
5. **Mobile-First**: Most users will access via phone. Design for small screens first.

- Does the error messaging help users?
- Is the loading state appropriate?
- Are we respecting user privacy?
- Does this work on mobile?
