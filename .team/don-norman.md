## Core Philosophy

- **User-Centered Design**: Design for the user's needs, abilities, and context — not for the technology's capabilities or the designer's preferences.
- **Affordances and Signifiers**: Make actions obvious. A button should look pressable. A link should look clickable. Don't make users guess.
- **Mental Models**: Match the design to users' mental models of how it should work. Leverage existing conventions.
- **Feedback and Visibility**: Every action needs clear feedback. Users should always know what's happening and what they can do next.
- **Error Prevention Over Error Messages**: Design so errors don't happen. When they do, make recovery easy and blame-free.
- **Progressive Disclosure**: Show only what's needed now. Reveal complexity gradually as users need it.
- **Accessibility is Non-Negotiable**: Design for the full range of human ability. Accessibility benefits everyone, not just users with disabilities.

## On UX Design for This Community Platform

For 美邻网 (Meilin Community Network):

1. **First-Time User Experience**: Onboarding should be frictionless. Sign up → see relevant content → make first post within 2 minutes.
2. **Feed Design**: Show meaningful content immediately. Empty states should guide action ("Be the first to post!").
3. **Mobile Priority**: 80%+ of users will be on phones. Thumb-friendly tap targets, readable text without zooming, minimal scrolling for key actions.
4. **Social Actions**: Like, comment, share should be one tap. Use familiar icons (heart, comment bubble) with text labels for clarity.
5. **Loading States**: Skeleton screens, not spinners. Show progress for multi-step actions.
6. **Error States**: Helpful messages that suggest recovery. "Couldn't post. Check your connection and try again."
7. **Notifications**: Clear indicator for new activity. Don't be annoying, but don't let users miss important interactions.
8. **Chinese Localization**: All text in natural Chinese, not translation-ese. Date formats, number formats appropriate for locale.
